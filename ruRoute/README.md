# ruRoute

**Умный split-routing для VPN серверов**

Российские сервисы идут напрямую через российский IP, весь остальной трафик — через зарубежный сервер. Клиентам ничего менять не нужно — всё происходит на стороне сервера автоматически.

---

## Как это работает

```
Клиент (любой протокол)
        ↓
  Российский VPS
        ↓
   ┌────┴────┐
   ▼         ▼
Госуслуги  YouTube
Сбер       Instagram
ВКонтакте  и т.д.
   ↓         ↓
Российский  Зарубежный
   IP          VPS
```

Клиент видит один и тот же сервер с одним и тем же ключом. Ключи и настройки менять не нужно.

## Установка

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ruRoute/ruRoute/main/install.sh)
```

Скрипт сам:
- Определит какая панель установлена
- Покажет список outbound тегов и попросит выбрать нужный
- Скачает базы доменов и IP
- Настроит автоматическое обновление баз каждую ночь в 3:00
- Установит watcher который восстанавливает rules если панель перезаписала конфиг

---

## Обновление баз вручную

```bash
bash /usr/local/ruRoute/update-lists.sh
```

## Базы доменов

Используется два источника:

**Ручная база** (`lists/domains-manual.txt`) — топ российских сервисов: госуслуги, банки, маркетплейсы, соцсети, стриминг и т.д. Обновляется мейнтейнером.

**Автобаза** — домены и IP-диапазоны из [antifilter.download](https://antifilter.download), обновляются автоматически каждую ночь.

Также используется встроенный `geoip:ru` — все IP-адреса российских AS автоматически идут напрямую.
